What we're building

A decentralized social superapp on Sui

ReeF (codename umbra in code) is a Farcaster/Base-app-style social network where identity, the social graph, content, payments and markets are native Sui primitives rather than rows in a company's database. The product is built for mainstream, mobile-first users — Lagos-first — so walletless onboarding and gasless UX are hard requirements, not nice-to-haves.

🔑

Walletless sign-in

Continue with Google → zkLogin (Enoki) mints a self-custodial Sui address. No seed phrases.

⛽

Gasless by default

Every action — post, follow, like, trade — is sponsored via Enoki. Users never hold gas.

🪸

You own it

Profiles, posts and follows are owned Sui objects — portable & censorship-resistant.

Architecture

Every capability maps to a Sui primitive

Durable, valuable state lives on-chain. High-frequency, low-value actions (likes, reposts, bookmarks) are off-chain and aggregated by an indexer — deliberately avoiding on-chain state-bloat.

On-chain model

Owned-object-first for parallelism

Why owned objects

Profile, Post and FollowSet are owned by the acting user. Owned-object transactions skip consensus and run in parallel — two users posting or following never contend. The only shared object is Registry (global handle/profile uniqueness), touched only on cold paths.

Reactions stay off-chain

Likes / reposts / bookmarks are signed messages, not objects — the indexer verifies the signature and aggregates counts. The stable post_id keeps on-chain settlement open for later.
